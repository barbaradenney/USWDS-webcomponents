## üèóÔ∏è Architecture Principles

### 1. USWDS-First Methodology
Every component follows the **USWDS-First** approach:
- Use official USWDS CSS classes directly (no custom styling)
- Load USWDS JavaScript globally via script tag pattern
- Let USWDS handle all transformations and behavior
- Components render HTML, USWDS enhances automatically

### 2. Light DOM Pattern
All components use **Light DOM** (no Shadow DOM) to ensure:
- USWDS CSS applies correctly without complex style injection
- Maximum compatibility with existing USWDS implementations
- Screen readers and assistive technologies work seamlessly
- Standard DOM manipulation and event handling

### 3. Script Tag Pattern (Mandatory)
**Global USWDS Loading:**
```html
<!-- .storybook/preview-head.html -->
<script src="/node_modules/@uswds/uswds/dist/js/uswds.min.js"></script>
```

**Component Implementation:**
```typescript
override firstUpdated(changedProperties: Map<string, any>) {
  // ARCHITECTURE: Script Tag Pattern
  // USWDS is loaded globally via script tag
  // Components just render HTML - USWDS enhances automatically

  super.firstUpdated(changedProperties);
  this.initializeUSWDSComponent();
}
```

### 4. Progressive Enhancement
Components follow web standards:
1. **HTML Layer** - Semantic markup works without JavaScript
2. **CSS Layer** - USWDS styles provide visual design
3. **JavaScript Layer** - USWDS enhances with interactive behavior
4. **Web Component Layer** - Adds encapsulation and reusability
